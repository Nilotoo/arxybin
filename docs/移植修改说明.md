## 逐条修改详情（含还原方法）

---

### 修改 ①：MSVC 运行时 → Windows 专属

**为什么改：** macOS 用 Apple Clang 编译器，不认识 `CMAKE_MSVC_RUNTIME_LIBRARY`，不包起来会直接报错。

**改之前：**
```cmake
set(CMAKE_MSVC_RUNTIME_LIBRARY "MultiThreaded$<$<CONFIG:Debug>:Debug>")
```

**改之后：**
```cmake
if(MSVC)
    set(CMAKE_MSVC_RUNTIME_LIBRARY "MultiThreaded$<$<CONFIG:Debug>:Debug>")
endif()
```

**还原方法：** 删除 `if(MSVC)` 和 `endif()`，把 `set(...)` 行顶回左边（去缩进）。

---

### 修改 ②：JUCE 路径 → 可配置

**为什么改：** macOS / CI 上没有 `C:/Users/Administrator/JUCE` 这个路径。

**改之前：**
```cmake
add_subdirectory(C:/Users/Administrator/JUCE
    ${CMAKE_BINARY_DIR}/JUCE)
```

**改之后：**
```cmake
if(NOT DEFINED JUCE_PATH)
    if(DEFINED ENV{JUCE_PATH})
        set(JUCE_PATH "$ENV{JUCE_PATH}")
    elseif(WIN32)
        set(JUCE_PATH "C:/Users/Administrator/JUCE")
    else()
        set(JUCE_PATH "$ENV{HOME}/JUCE")
    endif()
endif()
message(STATUS "JUCE_PATH = ${JUCE_PATH}")
add_subdirectory(${JUCE_PATH} ${CMAKE_BINARY_DIR}/JUCE)
```

**查找优先级（先匹配到的生效）：**
1. `cmake -DJUCE_PATH=/path/to/JUCE` 命令行指定的路径
2. 系统环境变量 `JUCE_PATH`
3. 平台默认值（Windows → `C:/Users/Administrator/JUCE`，其他 → `~/JUCE`）

**还原方法：** 删掉 `if(NOT DEFINED JUCE_PATH)...endif()` 整段和 `message(...)`，改回原来的 `add_subdirectory(...)`。

---

### 修改 ③：.rc 资源文件 → Windows 专属

**为什么改：** `.rc` 是 Windows 资源编译格式，macOS/Linux 上无法编译。

**改之前：**
```cmake
        Source/arxybin.rc
```

**改之后：**
```cmake
        $<$<PLATFORM_ID:Windows>:Source/arxybin.rc>
```

> `$<$<PLATFORM_ID:Windows>:...>` 是 CMake 生成器表达式，仅在 Windows 平台上展开内容。

**还原方法：** 把 `$<$<PLATFORM_ID:Windows>:Source/arxybin.rc>` 改回 `Source/arxybin.rc`。

---

### 修改 ④：macOS 加 AudioUnit 格式

**为什么改：** macOS 上的 Logic Pro 只用 AU 格式，不加就没法在 Logic 里用。

**改之前：**
```cmake
juce_add_plugin(arxybin
    ...
    FORMATS                  VST3 Standalone
    ...
)
```

**改之后：**
```cmake
if(APPLE)
    set(ARXYBIN_FORMATS VST3 Standalone AudioUnit)
else()
    set(ARXYBIN_FORMATS VST3 Standalone)
endif()

juce_add_plugin(arxybin
    ...
    FORMATS                  ${ARXYBIN_FORMATS}
    ...
)
```

> ⚠️ **踩坑记录：** 最初用 `$<$<PLATFORM_ID:Apple>:AudioUnit>` 生成器表达式直接写在 `juce_add_plugin()` 里，但 JUCE 函数内部解析 `FORMATS` 时不会展开生成器表达式，导致 AU 没编译出来。改成 `if(APPLE) set(...)` 变量的方式才生效。

**还原方法：** 删掉 `if(APPLE)...endif()` 段落，把 `FORMATS ${ARXYBIN_FORMATS}` 改回 `FORMATS VST3 Standalone`。

## CI 工作流说明

### `.github/workflows/build.yml`

每次推送代码到 GitHub 自动触发，同时也支持网页手动触发（Actions 页 → "Run workflow"）。

**包含三个 Job：**

| Job | 编译平台 | 产物 |
|-----|---------|------|
| `build-macos` | macOS (arm64) | VST3 + AudioUnit + Standalone + .pkg 安装包 |
| `build-windows` | Windows (x64) | VST3 + Standalone |
| `build-linux` | Ubuntu (x64) | VST3 + Standalone |

**macOS 构建流程：**
1. Clone JUCE 8.0.13 → `~/JUCE`
2. CMake 配置（Xcode 生成器）
3. Xcode 编译 Release
4. 打包 .pkg 安装包（使用系统自带 `pkgbuild`）
5. 上传所有产物

**产物内容（`arxybin-macOS.zip` 解压后）：**
```
arxybin..vst3/              ← VST3 插件
arxybin..component/         ← AudioUnit 插件
arxybin..app/               ← 独立运行程序
arxybin_v1.0.0_macOS.pkg    ← 一键安装包（推荐分发用这个）
```

**Linux 构建流程：**
1. `apt install` JUCE 依赖（ALSA / X11 / Fontconfig / cURL）
2. Clone JUCE 8.0.13
3. CMake 配置（Unix Makefiles）
4. make Release
5. 上传 VST3 + Standalone

**产物内容（`arxybin-Linux.zip` 解压后）：**
```
arxybin..vst3/              ← Linux VST3 插件
arxybin                      ← 独立运行程序（无扩展名）
```

### `.gitignore`

排除以下内容不上传 Git：
- `build/` — CMake 构建产物（几百 MB）
- `.claude/` — Claude Code 内部数据
- `.vscode/`、`.DS_Store` — IDE/OS 杂项
- `release/*.exe`、`release/*.zip` — 安装包二进制

## 给 Mac 用户的分发说明

### 方式一：.pkg 安装包（推荐）

1. 下载 `arxybin_v1.0.0_macOS.pkg`
2. 右键点击 → 按住 `Option` 键 → 选"打开"
3. 按提示完成安装
4. 打开 DAW 即可在插件列表中找到 "arxybin."

安装位置（自动）：
```
/Library/Audio/Plug-Ins/VST3/arxybin..vst3
/Library/Audio/Plug-Ins/Components/arxybin..component
/Applications/arxybin..app
```

### 方式二：手动安装

VST3 → 拖到 `/Library/Audio/Plug-Ins/VST3/`
AU → 拖到 `/Library/Audio/Plug-Ins/Components/`
App → 拖到 `/Applications/`

---

## 给 Linux 用户的分发说明

### 安装 VST3

把 `arxybin..vst3/` 整个文件夹复制到：

```
~/.vst3/
```

如果系统装了多个 DAW 想全局安装：

```
/usr/local/lib/vst3/
```

### 安装 Standalone

直接运行即可（需先 `chmod +x`）。

### Linux 依赖

首次运行如果报错缺少库，安装以下包（Ubuntu/Debian）：

```bash
sudo apt install libasound2 libx11-6 libfreetype6 libcurl4
```

## 已知限制

1. **未签名** — .pkg 和插件没有 Apple 开发者签名，安装时会提示"无法验证开发者"，需要右键+Option 打开
2. **仅 arm64** — CI 编译的是 Apple Silicon 版本，Intel Mac 用户需要额外编译 x86_64（可后续加 Universal Binary）
3. **Windows CI 偶发失败** — 可能是 `windows-latest` runner 上 CMake 配置问题，本地 VS 编译正常
